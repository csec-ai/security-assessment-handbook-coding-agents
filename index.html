Introduction
Background and context for this security assessment playbook.
Agentic AI in Software Development

Artificial intelligence has evolved at an unprecedented pace. In the span of a few years, large language models have moved from research curiosities to production-grade tools embedded in the daily workflows of software engineers worldwide. AI coding assistants — tools like GitHub Copilot, Cursor, Windsurf, and Claude Code — are now used to write, review, and refactor code across millions of codebases.

The latest generation of these tools goes beyond simple code completion. Agentic AI systems operate with a degree of autonomy that previous software generations did not — they plan multi-step tasks, invoke external tools, read and write files, execute terminal commands, and interact with remote services, all with minimal human intervention per action. This shift from passive assistant to active agent fundamentally changes the risk profile.

Traditional cybersecurity frameworks were built around securing deterministic software: defined inputs, predictable code paths, auditable state. Agentic AI introduces a new paradigm — one where the system's behaviour is shaped by natural language instructions that can originate from many sources, including attacker-controlled ones.

Novel Attack Surfaces

The attack surfaces introduced by agentic AI coding assistants are qualitatively different from those in traditional software security. The LLM context window — the stream of text the model uses to reason and act — becomes the primary attack surface. Any content that flows into that context, whether from a repository file, an MCP server tool response, a documentation page, or a configuration file, can potentially carry adversarial instructions.

This gives rise to a class of attacks with no direct analogue in conventional security: prompt injection, where malicious instructions are embedded in content the agent is expected to process, and supply chain compromise via AI context, where trusted tools or repositories are weaponised to influence agent behaviour rather than exploit binary vulnerabilities.

The extensibility of these systems — through plugin ecosystems, Model Context Protocol (MCP) servers, instruction files, skills, and lifecycle hooks — creates multiple trust boundaries, each of which is a potential entry point. Securing agentic AI requires reasoning about trust at each of these boundaries in a way that conventional application security does not.

About This Playbook

This playbook was developed by the Cyber Security Agency of Singapore to support security practitioners, developers, and risk owners in assessing and mitigating the security risks of AI coding assistant deployments.

The scope is deliberately focused on AI coding assistants — tools like GitHub Copilot, Cursor, Windsurf, and Claude Code — as they represent one of the most widely deployed categories of agentic AI in enterprise environments today. This focus allows the trust boundaries, attack vectors, and mitigations described here to be concrete and directly actionable, rather than generic to agentic AI at large.

It provides a structured, interactive playbook covering the system architecture of common AI coding assistant deployments, the attack vectors that have been identified, adversarial attack paths mapped to MITRE ATLAS tactics and techniques, and the mitigations applicable at each trust boundary.

Identified Attack Vectors

Seven attack vectors have been identified across IDE-based and CLI-based agentic coding assistant deployments. Each is detailed in the Attack Path tab with step-by-step adversarial chains and applicable mitigations.

T-001
Instruction file with malicious prompts
Malicious prompts embedded in instruction files (e.g. .cursorrules, CLAUDE.md) in cloned repositories direct the agent to execute attacker-defined actions. Applies to IDE and CLI agents.
T-002a
Malicious remote MCP server
A spoofed or attacker-controlled remote MCP server is configured in place of a legitimate one, injecting malicious tool responses into the agent context. Applies to IDE and CLI agents.
T-002b
Compromise of benign remote MCP server
A legitimate, already-trusted remote MCP server is compromised by an attacker who embeds malicious behaviour into its tools — exposing all users without any configuration change on their part. Applies to IDE and CLI agents.
T-003
Compromised local MCP server
A trojanised local MCP server package exfiltrates sensitive files when the coding assistant grants it filesystem access during project initialisation. Applies to IDE and CLI agents.
T-004
Malicious prompts in information sources
Prompt injection payloads embedded in external content (e.g. GitHub Issues, documentation) are fetched by the agent and executed as instructions — creating malicious hooks or exfiltrating data. Applies to IDE and CLI agents.
T-005
Skills with malicious instructions
Malicious skill packages published to public repositories embed adversarial instructions that, when loaded into Claude Code, direct the agent to exfiltrate credentials or establish persistence. CLI agents only.
T-006
Hooks with malicious actions
Repositories pre-configured with malicious lifecycle hooks in settings.json automatically execute attacker payloads when a user clones the repo and triggers an agent session event. CLI agents only.
